索引

对于任何的数据库管理系统，索引都是进行优化的最主要因素，对于少量的数据，即使没有合适的索引对数据库性能的影响并不是很大，但是随着数据量的增加，数据库性能会急剧下降，所以索引的目的在于提高检索数据的效率。

索引占用空间么？

索引占用空间，但是空间小；能够带来速度的明显提升

索引是不是越多越好？

不是，索引也占用空间，多个索引就占用更多空间，给经常需要用到的内容建立索引，否则会查询建立了索引，占用了空间，但是很少使用，同时索引会提高查询的速度，但是会降低添加，更新，删除的速度（不仅操作数据库表，也要操作索引）

一般给哪些列建立索引

数据库会自动给主键和唯一键，外键建立索引，要给经常出现在where字句中的或者order by 子句中的列建立索引

索引类型

单列索引和多列索引 sal | sal hiredate

唯一索引和非唯一索引 empno | sal deptno

存储结构：B-Tree,R-Tree和Hash,其中B-Tree是最流行的存储结构，一般也是默认的存储结构。

索引的使用：

—查询所有的数据（没有必要使用索引）

select * from emp

—查询指定编号的索引（使用了索引，自动给主键创建所有）

select * from emp where empno=7839 只要条件中出现了索引列，就会用索引

—查看表的索引

show index from emp; 自动给主键，外键，唯一键创建了索引

—查询指定名称的员工信息(逐个查询效率低下，没走索引)

select * from emp where ename=‘SCOTT’

解决：给ename 列创建索引

create index index_emp_ename on emp(ename),这个时候会用索引

—删除索引

drop index index_emp_enam e on emp

—创建索引，给ename 列创建索引，降序排列

alter table emp add index index_emp _ename(ename desc)

—索引要修改，先删除再新建即可。
