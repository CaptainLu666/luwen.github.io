1.查看资源占用：

kubectl describe node “cn-beijing.172.16.89.154” | sed ‘1,/Non-terminated Pods/d’

2.进入pod

Kubectl exec -it pod -n namespace /bin/bash

3.查看pod资源消耗

Kubectl top pod -n prod

4.kubectl get nodes –show-labels

5.kubectl get nodes

4.删除evicted状态pod

kubectl get pods | grep Evicted | awk ‘{print $1}’ | xargs kubectl delete pod

指定 Node 节点调度

有三种方式指定 Pod 只运行在指定的 Node 节点上

  • nodeSelector：只调度到匹配指定 label 的 Node 上
  • nodeAffinity：功能更丰富的 Node 选择器，比如支持集合操作
  • podAffinity：调度到满足条件的 Pod 所在的 Node 上

kubelet配置策略

–eviction-soft=memory.available<1Gi

–eviction-soft=memory.available<10%

–system-reserved=memory=2Gi

–eviction-soft-grace-period=1m30s

–eviction-hard=memory.available<500Mi –system-reserved=memory=2Gi

–eviction-minimum-reclaim=memory.available=2Gi

–eviction-hard=memory.available<500Mi,nodefs.available<1Gi,imagefs.available<100Gi –eviction-minimum-reclaim=“memory.available=0Mi,nodefs.available=500Mi,imagefs.available=2Gi”`
